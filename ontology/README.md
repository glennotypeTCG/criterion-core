# Criterion Core Ontology

## Purpose

This folder contains the reusable system primitives for Criterion-related work.

Narrative files explain the system. Ontology files govern the system.

Use this folder when a human or AI agent needs the stable logic behind the work rather than a long-form memo, pitch, or marketing artifact.

## Current Ontology Types

| Type | Folder | Purpose |
|---|---|---|
| What Has to Be True | `what-has-to-be-true/` | Defines the core belief conditions required for each entity and stakeholder relationship to work. |

## Naming Convention

Ontology primitive files use this pattern:

`<entity>.<stakeholder>.md`

Examples:

- `fetchsignal.idea-principal.md`
- `swiftvc.lp.md`
- `axiomfoundry.idea-principal.md`

The folder defines the primitive type. The file name defines the entity and stakeholder.

## Repo Rule

Do not treat the ontology files as copy drafts.

They are source-of-truth primitives that should inform copy, product flows, pitch materials, investor narratives, founder conversations, and future MCP or agent workflows.

When messaging drifts, return to the ontology first.
