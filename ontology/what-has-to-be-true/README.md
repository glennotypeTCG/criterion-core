# What Has to Be True Primitives

## Purpose

These files define the core belief conditions required for each part of the system to work.

They are not marketing pages.

They are ontology primitives used to keep:

- Messaging
- Strategy
- Product design
- Founder conversations
- LP conversations
- Capital structure
- Formation logic
- AI agents and MCP workflows

aligned around the same underlying beliefs.

## Current Primitives

| Entity | Stakeholder | File | Core Belief |
|---|---|---|---|
| FetchSignal | Idea Principal | `fetchsignal.idea-principal.md` | The Idea Principal trusts FetchSignal to safely produce credible market signal. |
| SwiftVC | LP | `swiftvc.lp.md` | LPs believe early signal can improve formation decisions and venture outcomes. |
| Axiom Foundry | Idea Principal | `axiomfoundry.idea-principal.md` | The Idea Principal believes Axiom can help create a more valuable company than they could create alone. |

## System Relationship

FetchSignal makes the idea safe to test.

Axiom Foundry makes the idea possible to form.

SwiftVC makes the formed company worth funding early.

## Standard Sections

Each primitive should contain:

- Ontology Type
- Entity
- Primary Stakeholder
- Canonical Belief
- Plain-English Version
- Emotional Job
- Functional Job
- Proof Required
- Failure Mode
- Messaging Guardrails
- Related Primitives
