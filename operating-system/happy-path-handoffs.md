# Canonical Happy Path Handoffs

## Purpose

This file defines the default positive-signal path through the Criterion ecosystem. It describes how an idea person moves from market validation to first-check capital, company formation, independent spinout, and long-tail portfolio management.

## Summary Chain

```text
Idea Person
→ Fetch Happens C6
→ Fetch Happens C7
→ Positive Signal
→ SwiftVC C6
→ Term Sheet Intent
→ Axiom Foundry C6
→ Dual Signature
→ SwiftVC C7 Long-Tail
→ Axiom Foundry C7 Formation Sprint
→ NewCo Graduation
→ Independent Startup
→ Raise, grow, fail, or exit
```

## Handoff 1: Idea Person → Fetch Happens C6

| Field | Definition |
|---|---|
| Trigger | Idea person has a concept worth testing. |
| Owner | Fetch Happens |
| Customer State Before | Idea person with unvalidated concept. |
| Customer State After | Validation customer. |
| Artifact | Validation engagement or study scope. |
| Decision | Should Fetch accept and sell a validation study? |
| Success Standard | Idea person commits to a structured validation process. |

## Handoff 2: Fetch Happens C6 → Fetch Happens C7

| Field | Definition |
|---|---|
| Trigger | Validation study is sold or authorized. |
| Owner | Fetch Happens |
| Artifact | Study plan and evidence plan. |
| Decision | What must be tested to know whether customers want it? |
| Success Standard | Study launches with clear target customer, problem hypothesis, and signal criteria. |

## Handoff 3: Fetch Happens C7 → SwiftVC C6

| Field | Definition |
|---|---|
| Trigger | Fetch completes study and finds positive signal. |
| Owner Before | Fetch Happens |
| Owner After | SwiftVC investment screen |
| Artifact | Signal Dossier |
| Decision | Should SwiftVC evaluate the opportunity for first-check capital? |
| Success Standard | SwiftVC has enough evidence to begin investment screening. |

A positive Fetch signal does not automatically create a term sheet. It triggers SwiftVC evaluation.

## Handoff 4: SwiftVC C6 → Axiom Foundry C6

| Field | Definition |
|---|---|
| Trigger | SwiftVC reaches term sheet intent. |
| Owner Before | SwiftVC |
| Owner After | SwiftVC and Axiom jointly own conversion into the combined formation package. |
| Artifact | Investment Memo and Formation Readiness Packet. |
| Decision | Can this opportunity be responsibly offered capital plus formation services? |
| Success Standard | Axiom confirms formation scope, capacity, timeline, and fit. |

Axiom should be involved before final term sheet delivery when the offer depends on Foundry services.

## Handoff 5: Axiom Foundry C6 → Dual Signature

| Field | Definition |
|---|---|
| Trigger | Joint offer is ready. |
| Owner | SwiftVC and Axiom Foundry |
| Artifact | Joint Offer Packet, investment documents, services agreement. |
| Decision | Will the idea person accept the combined capital and formation package? |
| Success Standard | Investment documents and services agreement are both signed. |

## Handoff 6: Dual Signature → SwiftVC C7 and Axiom Foundry C7

| Field | SwiftVC C7 | Axiom Foundry C7 |
|---|---|---|
| Trigger | Investment documents signed. | Services agreement signed. |
| Timeline | Long-tail venture lifecycle. | Target roughly six-month formation sprint. |
| Primary Work | Portfolio management, ownership retention, reserves, follow-on support, LP reporting. | Formation delivery, product/GTM planning, operating cadence, spinout readiness. |
| Success Standard | Markup, follow-on, liquidity, exit, or disciplined failure. | Independent NewCo ready to operate outside the Foundry. |

## Handoff 7: Axiom Foundry C7 → Independent NewCo

| Field | Definition |
|---|---|
| Trigger | Formation milestones are complete. |
| Owner Before | Axiom Foundry |
| Owner After | NewCo leadership, with SwiftVC remaining economically exposed. |
| Artifact | Graduation Packet |
| Decision | Is the NewCo ready to leave the Foundry and operate independently? |
| Success Standard | The NewCo has enough product, market, operating, and fundraising structure to grow in the wild. |

## Required Handoff Artifacts

| Artifact | Owner | Purpose |
|---|---|---|
| Signal Dossier | Fetch Happens | Captures market evidence and recommendation. |
| Investment Memo | SwiftVC | Captures investment thesis, risks, ownership logic, and decision. |
| Formation Plan | Axiom Foundry | Captures how the company will be formed. |
| Joint Offer Packet | SwiftVC + Axiom | Presents the combined capital and formation package. |
| Graduation Packet | Axiom Foundry | Transfers operating ownership to the independent NewCo. |
| Portfolio Update | SwiftVC | Tracks long-tail investment progress. |

## Guardrail

No one entity should be able to force the others downstream. Positive signal, investment readiness, and formation readiness must each be true before a combined offer is made.
