# Canonical Happy Path Handoffs

## Purpose

This file defines the default positive-signal path through the Criterion ecosystem.

## Summary Chain

```text
Idea Person
-> Fetch Happens C6
-> Fetch Happens C7
-> Positive Signal
-> SwiftVC C6
-> Term Sheet Intent
-> Axiom Foundry C6
-> Dual Signature
-> SwiftVC C7 Long-Tail
-> Axiom Foundry C7 Formation Sprint
-> NewCo Graduation
-> Independent Startup
```

## Handoff Rules

1. A positive Fetch signal triggers SwiftVC evaluation. It does not automatically create a term sheet.
2. SwiftVC term sheet intent should trigger Axiom formation scoping before a final combined offer is made.
3. Axiom should confirm formation scope, timeline, capacity, and fit before the opportunity is sold as a Foundry-backed formation package.
4. The combined close requires both investment documents and the Foundry services agreement.
5. SwiftVC C7 begins at investment close and runs through the long-tail portfolio lifecycle.
6. Axiom Foundry C7 begins at services close and ends when the NewCo is ready to operate independently.

## Required Handoff Artifacts

| Handoff | Owner | Required Artifact |
|---|---|---|
| Fetch C7 to SwiftVC C6 | Fetch Happens | Signal Dossier |
| SwiftVC C6 to Axiom C6 | SwiftVC | Investment Memo and Formation Readiness Packet |
| Axiom C6 to Dual Signature | Axiom Foundry and SwiftVC | Joint Offer Packet |
| Dual Signature to Delivery | SwiftVC and Axiom Foundry | Investment Documents and Services Agreement |
| Axiom C7 to Independent NewCo | Axiom Foundry | Graduation Packet |

## Guardrail

No one entity should be able to force the others downstream. Positive signal, investment readiness, and formation readiness must each be true before a combined offer is made.
